### 方法的重载

1. 同一个类，方法名相同，参数不同（类型不同、参数不同、顺序不同），与返回值无关

### 方法的内存

#### java内存分配

- 栈			方法运行时使用的内存，方法进栈运行，运行完毕就出栈
- 堆			new出来的，都在堆内存中开辟了一个小空间
- 方法区		存储可以运行的class文件
- 本地方法栈	JVM在使用操作系统功能的时候时候
- 寄存器		给CPU使用

### 构造方法

构造方法/构造器/构造函数

```java
public class 类名{

	修饰符 方法名(参数){
	方法体
	}
	
}
```

1. 方法名与类名完全相同
2. 没有返回值类型，没有返回值
3. 创建对象的时候由虚拟机调用，不能手动调用构造方法
4. 每创建一次对象，就会调用一次构造方法
5. 创造对象的时候，虚拟机会自动调用构造方法给成员变量进行初始化

### String

java.lang.String
字符串不可变，它们的值在创建之后不能被更改

内存原理：
栈内存：方法运行时进栈，执行完毕出栈
堆内存：new出来的对象都在这里
方法区：字节码文件 临时存储

StringTable（串池）：当使用双引号直接赋值时，系统会检查该字符串在串池中是否存在。
					若不存在，创建新的，若存在，复用。

==比较：基本数据类型比较的是数据值，引用数据类型比较的是地址值。

### StringBuilder

作用：提高字符串的操作效率
StringBuilder可以看作一个容器，创建之后里面的内容是可以变的
java在底层对StringBuilder进行了特殊处理，所以打印对象不是地址值而是属性值

链式编程：
当调用一个方法时，不需要用变量接收他的结果，可以继续调用其他方法
